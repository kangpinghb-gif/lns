# UI 页面功能与内容说明

## 1. 目标

将当前 UI 拆分为多页面结构，并统一到新的 LNS 品牌视觉，形成清晰用户路径：

`Home -> Input -> Kline -> Analysis -> YearDetail`

目标不是做成单纯的展示页，而是让用户按步骤完成：

1. 输入个人信息
2. 查看人生趋势总览
3. 理解基础命理判断
4. 深入查看某一年的流年详情

---

## 2. 页面总览

| 页面 | 作用 | 核心内容 | 核心交互 |
|---|---|---|---|
| Page 0: Home | 品牌首页 | Logo、价值主张、趋势预览、生成入口 | 跳转输入页、直接看样例 |
| Page 1: Input | 天命接入页 | 收集用户基础命理信息 | 填写、校验、提交 |
| Page 2: Kline | 运势大盘页 | 展示命运指数走势图、当前状态 | 查看趋势、点击年份、进入解析 |
| Page 3: Analysis | 命理透视页 | 展示五行能量分布与核心性格向量 | 查看命理解读、返回总览 |
| Page 4: YearDetail | 流年详情页 | 展示某一年的事业/财运/健康/感情/宜忌 | 切换年份、返回总览 |

---

## 3. Page 0: Home

### 页面目标

统一品牌入口，让用户先建立“LNS 是什么”和“这套系统解决什么问题”的认知。

### 页面内容

- LNS 图标与横版字标
- 品牌主标题
- 一句话价值说明
- 趋势图预览
- 生成入口按钮
- 页面结构说明卡

### 主要交互

- 点击开始生成，进入 `Page 1: Input`
- 点击结构样例，进入 `Page 2: Kline`

---

## 4. Page 1: Input

### 页面目标

收集用户进入系统所需的基础个人信息，尽量降低首屏阻力。

### 页面内容

- 品牌名 / 产品名
- 一句话说明
- 输入表单
- 开始按钮
- 简短说明

### 建议字段

必填：

- 姓名
-  性别
- 出生日期
- 出生时间
- 出生地


### 主要交互

- 填写信息
- 字段校验
- 点击开始
- 成功后进入 `Page 2: Kline`

### 备注

- 输入日期用现有的日期选择期（要有农历和公历选择）

---

## 5. Page 2: Kline

### 页面目标

让用户第一眼看到“结果感”，先理解整体趋势。

### 页面内容

- 页面标题
- 动态 K 线图
- 当前阶段
- 能量值
- 风险值
- 当前大运 / 流年提示
- 命局摘要
- 进入解析按钮

### 核心模块

#### 4.1 K 线图区域

- Canvas 自绘趋势图
- 蜡烛线 / 折线 / 网格背景
-  x值以5年为一个刻度；y值以25分为一个刻度
- 支持 60 年维度的数据表达
- 点击zuo biao

#### 4.2 状态指标区

- 当前阶段
- 能量值
- 风险值
- 当前趋势说明

#### 4.3 命局摘要区

- 日主
- 十神落位
- 五行旺衰
- 当前大运主题

### 主要交互

- 点击年份
- 切换时间范围
- 点击“查看解析”
- 点击某年进入 `Page 4: YearDetail`

### 备注

这是主结果页，不要塞太多解释文本。

---

## 6. Page 3: Analysis

### 页面目标

解释为什么会出现当前趋势，给出基础命理判断。

### 页面内容

- 命局结构
- 五行能量分布
- 十神落位
- 性格主轴
- 行事倾向
- 用神倾向
- 当前命理解读

### 核心模块

#### 5.1 命局结构区

- 日主
- 四柱
- 十神结构
- 命局核心关系

#### 5.2 五行能量区

- 木火土金水分布
- 强弱对比
- 偏旺 / 偏弱提示

#### 5.3 性格与倾向区

- 核心性格向量
- 行事倾向
- 适合的处理方式

#### 5.4 命理解读区

- 当前判断摘要
- 简明结论
- 返回 K 线入口

### 主要交互

- 展开 / 收起解释
- 返回 `Page 2: Kline`
- 进入某一年详情

### 备注

这一页应承担“解释”，不是“重复展示结果”。

---

## 7. Page 4: YearDetail

### 页面目标

深度拆解某一年的流年内容。

### 页面内容

- 年份标题
- 年度总评
- 事业
- 财运
- 健康
- 感情
- 年度宜忌
- 年度风险与机会
- 与前后年份对比

### 核心模块

#### 6.1 年度总评

- 这一年整体好坏
- 是否适合推进关键事项
- 这一年的主基调

#### 6.2 分项拆解

- 事业
- 财运
- 健康
- 感情

#### 6.3 年度宜忌

- 宜做什么
- 忌做什么
- 风险提示

#### 6.4 年份切换

- 上一年
- 下一年
- 返回 K 线总览

### 主要交互

- 点击年份切换
- 上下年切换
- 返回 `Page 2: Kline`

### 备注

这一页适合做高密度内容页，但不能把所有年份都堆在这里。

---

## 8. 页面间关系

### 主流程

`Home -> Input -> Kline -> Analysis -> YearDetail`

### 辅流程

- `Home -> Input -> Kline -> YearDetail`
- `Kline -> Analysis -> Kline`
- `Analysis -> YearDetail -> Kline`

### 用户理解顺序

1. 先理解产品与品牌
2. 再填信息
3. 先看总览
4. 再看解释
5. 最后看年度细节

---

## 9. 全局内容收敛原则

- 首屏只保留一个核心视觉
- 结果先于解释
- 总览先于细节
- 多余字段后置
- 能用数据就不要堆长文
- Logo 使用遵循“一套品牌、两种用法”：小尺寸用 icon，大尺寸用横版字标
- 不能用一个页面同时承担所有职责

---

## 9. 当前页面字段建议

### 推荐保留

- 命局分析
- 五行旺衰
- 大运流年
- 状态分值
- 命局结构
- 用神倾向
- 年度详情

### 建议弱化或后置

- 工作单位
- 过多问卷字段
- 复杂说明文本
- 首屏过多的行动建议
